## 一、DHS 设计示意图与架构图理解

### 1.1 图示内容

**图1 - DHS 设计示意图**
- **(a) 标准射线采样**：相机发出射线，在 `f_near` 与 `f_far` 之间均匀分布灰色采样点。
- **(b) 深度层次采样 (DHS)**：
  - **Depth Map**：上方矩形框，蓝色箭头指向射线上一簇采样点。
  - **Global Samples**：大括号覆盖整条射线上的蓝色+橙色点，表示整体采样集合。
  - **Surface Samples**：橙色点密集分布在深度图指示的表面附近，旁有红色标注「采样范围公式」。
  - 含义：用深度先验在表面附近加密采样，同时保留全局稀疏采样覆盖背景/前景。

**图2 - 架构图**
- **输入**：`I_1`（多帧街景图）。
- **2D 提取**：DINOv2(冻结)→Feat_2D；DA3→Depths；Mask2Former→GT_S。
- **2D→3D**：2D-to-3D Lifting → Feat_3D。
- **语义密度场**：Feat_3D → Φσ(密度σ)、ΦS(语义S) → Semantic Density Field。
- **DHS**：输入 Depths、ΦS/S，输出参与体积渲染。
- **体积渲染**：得到 Render_S、Render_I；监督：L_sem(Render_S vs GT_S)、L_ph(Render_I vs I_1)。

### 1.2 相关/相似工作（图示与思路）

- **深度引导采样**：NeRF 原始 hierarchical sampling、Mip-NeRF 360、Depth-Guided NeRF (EMD)、Error-aware Adaptive Shells、AdaNeRF。
- **2D 特征提升到 3D**：Lift3D、LUDVIG、FiT3D（DINOv2→3D Gaussian）。
- **语义 NeRF / 场景补全**：Semantic-NeRF、SPNeRF、开放词表 3D 分割、S4C。
- **深度不确定性**：Bayes' Rays、Depth-Guided NeRF via EMD、U-NeRF。

### 1.3 图示可改进点（简要）

- 明确「采样范围公式」：例如写出 `[d−kσ, d+kσ]` 或注明基于深度不确定性。
- 区分 Global 与 Coarse：若蓝点为粗采样、橙点为表面精细采样，可在图注中说明。
- 标注 Depth Map 来源：如「DA3 单目深度」或「Coarse pass」。
- 架构图中标出 DHS 的输入输出与在渲染中的具体作用（控制采样位置/数量）。

### 2.3 近场 IoU 不如 S4C 的可能原因（当时分析）

1. **DA3 深度先验在近场的相对误差**：AbsRel~0.1 在近场（如 5m）对应约 0.5m 误差；采样范围 `[d−2σ, d+2σ]` 若过窄，容易错过真实表面。
2. **n_global_samples=0 的消融**：完全依赖深度先验，无全局兜底，对错误深度更敏感。
3. **逆深度采样的分配**：lindisp 虽在近场更密，但若深度先验偏了，采样仍会偏离表面；均匀采样在近场反而覆盖更均匀。
4. **表面采样范围**：σ(d)=max(0.5, 0.1×d)，近场范围有限，对 DA3 偏差容错不足。

（注：后续结论是优先采用 32+32 + lindisp=true，并保留 n_global=32，不做 n_global=0 的正式训练。）

## 三、提高近场 IoU 的优先级与具体设置

### 3.1 消融说明

n_global_samples=0 仅用于消融实验，正式训练不采用；以下建议均假设保留一定 global 采样。

### 3.2 优先级最高的训练设置

**结论：32+32 + 逆深度采样（lindisp=true）**

- **n_surface_samples**: 32  
- **n_global_samples**: 32（在 16/32/48 中优先 32，早期近场 IoU 最好）  
- **lindisp**: true（逆深度采样）  
- **absrel_prior**: 0.10  
- **min_thickness**: 0.5  

理由简述：
- 同期 13k 步时，32+32 + lindisp=true 近场 IoU@12.8=57.01，高于 24+40(53.20) 和 32+32 均匀(55.11)。
- 逆深度在远场 IoU@51.2 比均匀采样高约 7.7 点，全距离更均衡。
- 24+40 表面点少(24)、全局点多(40)，近场表面刻画不如 32+32。

### 3.3 global 选 16 / 32 / 48 哪种？

- **推荐 32**：与表面采样 32 对等，实验里近场表现最好。
- 16：全局更少，更依赖深度先验，鲁棒性略差。
- 48：全局过多，会稀释表面附近采样密度，近场可能变差。

### 3.4 均匀采样 vs 逆深度采样

- **推荐逆深度采样（lindisp=true）**。
- 均匀采样（lindisp=false）在近场 13k 步时略高(55.11 vs 51.11)，但该实验只训到 13k；且远场 IoU 明显更低(32.75 vs 40.40)，综合不如逆深度。
- 逆深度在近场 13k 步已达 57.01，长期训练更稳。

### 3.5 可选变体

- 更偏表面：`n_surface_samples: 40`, `n_global_samples: 24`。
- 扩大近场鲁棒性：`absrel_prior: 0.15`, `min_thickness: 0.8`。

### 4.2 从 54000 恢复还是从 53000 恢复？

**结论：从 `training_checkpoint_54000.pt` 恢复。**

- **training_checkpoint_54000.pt** 包含：模型权重、优化器状态（如 Adam 的动量）、学习率调度器状态、当前训练步数（54000）。恢复后训练无缝延续。
- **best_*_model_53000_*.pt** 一般为仅权重的保存，不含优化器与调度器。从 53000 恢复相当于用 53k 的权重「冷启动」，优化器重置，且会重新经历 53001→54000 这段更新，不利于稳定收敛。
- 53000 与 54000 的指标差异可视为正常波动，优先保证优化器与调度器的连续性。

## 六、优化器与 a1 / a1_lidar 指标说明

### 6.1 当前训练使用的优化器

- **优化器**：Adam（S4C/BehindTheScenes 惯例）。
- **学习率**：1e-4（来自 config，如 `learning_rate: 0.0001`）。
- **调度**：warmup + step decay（如 `step_size: 120000`, `gamma: 0.1`）。

### 6.2 为什么 a1 很小、a1_lidar 很大？

**a1 定义**：满足 \(\max(\frac{d_{pred}}{d_{gt}}, \frac{d_{gt}}{d_{pred}}) < 1.25\) 的像素占比。

**两者区别：**

| 项目 | a1 | a1_lidar |
|------|-----|----------|
| **使用的 GT** | 自监督/多视角重投影等（渲染深度或伪深度） | LiDAR 稀疏深度（真实测量） |
| **评估范围** | 全图所有像素（含天空、远距离、无真实深度的区域） | 仅 LiDAR 点投影到的像素（路面、建筑、车辆等） |

**a1 偏小的原因：**
- 在全图计算，包含天空、无穷远、遮挡等没有可靠深度 GT 的区域。
- 这些区域模型预测与「伪 GT」不一致，会拉低 a1。
- a1 更多反映「渲染深度与自监督信号的一致性」，不是绝对深度精度。

**a1_lidar 较大的原因：**
- 只在有 LiDAR 测量的像素上评估，多为有明确几何的区域。
- 这些区域正是 DA3 与自适应采样表现较好的范围，因此 a1_lidar 能到 0.87 左右，更能反映真实深度质量。

**结论**：a1 低属正常，不必以 a1 为主要优化目标；**以 a1_lidar 作为深度效果的主要指标**即可。
